Professional Portfolio: Frontend Development

Individual project: Create a professional portfolio using HTML, CSS, JavaScript and Bootstrap components. Throughout multiple phases, I also iterated on code responsive desing elements and added interactive details. It uses utility classes for typography and spacing to space content out within the larger container.

My Role: Front End Developer

Timeline: 5 weeks

Research

Problem Statement: With a very competitive and broad range of amazing designs on the internet, how might I, Lynlee Roeber, promote myself in UX/UI Design and stand out among the crowd?

Solution: I need to create an interactive webpage that demostrates the skills I have learned in frontend development and advocate my design thinking process whilte designing an athestically pleasing portfolio.

Action Plan: I developed a 60-120 action plan. This included bringing focus of how to accomplish my dreams and put in place a layout of my career expectations for my future endeavors.

Personal Style Guide: With infinite choices, my style tile was more challenging than I thought it would be. There are so many layers to my creativity, how can I only choose one path? Eventually, I narrowed it down and was happy with the end result. This eventually provided a foundation for my color scheme and logo design.

Iteration Phases

UX Design Portfolio and Job prep: After creating a 120 day action plan and job prep, the assigment helped implement the visual aspects of "who I am" as a designer. From Moodboard to personal style guide I was able to develop a responsive portfolio in Figma.

Code your Portfolio Homepage: While redlining the portfolio page in Figma, I had to consider the coding possibilities in both HTML and CSS. Creating a clone from the mockup in Figma deeded structure and styling in order to upload the visual webpage through GitHub.

Iteratation of the Portfolio and Code Responsive Design Elements: 3 Iterations were made to complete the final product. Originally, the font on the navbar enlarged when hovered over, but the enlarged font made the page shfit, so no elargement was the way to go. The sections of skills and work were too close togeather, therefore were split into 2 separate pages. A contact page was not designed in HW17 and was added per tester request.

Adding Interactive Elements: The interactive pieces were fun to make and included to create a sense of engagement for the user. When hovering over buttons, they enlarge and change color. The use of gifs also gave the website a feeling of fun and playfulness, while keeping a professional look.

Product

The end product created is a professional portfolio coded and designed for full website engagement. The UX/UI Designer, Lynlee Roeber, is represented well in establishing accomplishments, design projects, and Frontend development abilities.
